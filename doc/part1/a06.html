    PART I - 6회(15/01/02) 바벨피쉬 후기입니다.
    이번주 스터디 후기 도배 중인  바벨피시 어항관리자입니다.

    신년 첫 스터디 연휴 낀 불금임에도 불구하고
    많은 분이 참여해 주셨습니다.
    검색엔진에 이렇게 관심있는 분들이 많을 줄이야
    업무에도 연관 있는 스터디라 필히 참여하고 싶었습니다.

    elasticsearch 뭔가 했는데 검색엔진이라고 하길래
    Lucene 같은건가 싶었은데 Lucene 기반이네요
    대형검색 포털은 자체 엔진구축해서 만들어서 쓰겠지만
    검색엔진 필요하면 이거 가져다 쓰시면 됩니다.

    elasticsearch = Lucene + (분산,기타 장점 여러개)
    정도로 이해했습니다.
    shard 와 replica 를 염두해둔 구성으로
    분산 검색을 목적으로 만들어진것으로 보입니다.
    보통은 Lucene 두 대 정도 구성이면 충분합니다.

    노트북이 윈도우인지라 설치시 몇가지만 해주면됩니다.
    - java 1.7이상 필요.
    - java_home 설정필수. set java_home=c:\자바경로
    이 정도면 실행가능합니다. 엄청 엄청 간단하네요.

    사용해 보실려믄 2월까지 쓸 수 있는
     cloud.biopython.net 에도 설치가 되어 있습니다.
    http://cloud.biopython.net:60919/
    사용법은 발표자료 참고해서 이용해보세요.
    http://babelpish.github.io/
    보강 C. 오픈소스 검색엔진 - Elasticsearch 에 링크 있습니다.

    추가로 한글 검색용으로 쓸려면 은전 한 잎 플러그인 필요합니다.
    로그분석용으로 Free Splunk 스택 조합도 소개해주셨는데
    오픈 소스만 가지고 만든 꽤 멋진 구성으로 보입니다.

    서버 로그 분석을 아주 가끔 하긴하는데
    한 달 짜리 로그에서 뭐 찾을려면 보통 7~8시간씩 걸립니다.
    디스크 용향 큰 구형 서버 몇 대 분산 구성해서 서버로그 검색용으로
    만들고 marvel 추가해서 대시보드로 보면 좋겠다라는 생각도 듭니다.
    검색을 빠르게 할 수 있으면 질의어 잘 만들어서 의미있는 데이터를
    찾아볼 수 도 있겠습니다.
